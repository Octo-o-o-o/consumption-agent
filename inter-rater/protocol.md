## 1 Scope statement

**This study measures:** the inter-annotator reliability of the two-level criterion proposed in *Consumption Agents* §2.2, as operationalized by the accompanying codebook (published with this protocol), applied at the level of a single **act** (one run of a system and its output), under **given facts** (each case supplies a vignette and, for Level 2, a structured fact card).

**This study does not measure:** (a) the fact-acquisition layer — in real deployments a classifier must first find out audience, persistence, and funding facts; here they are supplied; (b) the act→product aggregation rule (a product is a distribution over acts; no threshold for "dominant acts" is tested); (c) system-level verdicts of the kind given in the paper's Table 2; (d) the *validity* of the criterion — reliability and validity are reported separately (§6.4), and agreement with the authors' answer key is labeled as evidence about correspondence with intended readings, never as reliability.

The instrument under test is the **codebook**, not an abstract criterion: following standard content-analysis methodology, reliability is a property of (instructions × coder qualifications × training), all three of which are reported.

## 2 Design overview

- **Unit:** the act. Every case is a vignette (80–140 words, plain English ~B2 level) describing one system performing one concrete run in one situation. No bare system names.
- **Case set (n = 50):** three strata —
  - **Representative stratum (22):** systematic sample from a pinned third-party public list of AI products (sampling frame recorded with version, date, URL before recruitment; every k-th entry; 15 items) plus a reviewer-nominated directed subset (7 items), one *typical act* per system;
  - **Anchor stratum (13):** simple production-side acts (7) plus six designed hybrid-gate cases (single run, two output components with destinations on opposite sides of the principal's sphere), added to balance category prevalence under revision R1 (see header note; the original "each category ≥20%" wording is superseded), sourced from common production-agent and mixed scenarios; disclosed as a designed stratum; never enters the co-primary endpoints;
  - **Adversarial stratum (15):** six boundary families × 2–3 **minimal pairs** (same act, one criterion-relevant fact flipped). Family list and vignettes in the case-set design document (ADV-12 narrowed to a single run with an act-delimiting sentence, 2026-07-13, so that its placement stays cleanly on the consumption side and the F5 loyalty flip is carried by the Task B fact card).
- **Two sessions per verification annotator, on different days:**
  - **Session A (Task A, all cases):** three output tests (audience / lifetime / information) recorded separately, then placement (production / consumption / hybrid, forced choice), confidence (1–5), an "insufficient information" flag (does not replace the forced choice); hybrid choices require a one-line component decomposition; adversarial-stratum items require a one-line rationale.
  - **Session B (Task B, a fixed 20-case subset designated by the researcher, identical for all annotators):** framed as "assume this act places on the consumption side"; judge the four traits (T1–T4) on a three-point ordinal scale (✓ satisfied / ◐ partially or degenerately satisfied / ✗ failed) from the vignette plus a structured **fact card** (fixed fields, publicly citable evidence only). Task B is released only after Task A is completed and locked. Task B reliability is reported as *conditional* reliability.
  - Each session ends with 5 repeated early items (surface order changed) serving as intra-rater consistency checks and attention checks. All items are timed.
- **Panels:**
  - **Verification panel:** 5 human annotators, cold-recruited (Prolific-class platform, technical-literacy screener), **exposure-screened** (anyone who has read the paper, the handbook, or the HN thread is excluded); paid by actual time (~3 hours across two sessions). Heterogeneous backgrounds (HCI / RecSys / engineering) are reported descriptively; no subgroup inference.
  - **Baseline panel:** 3 annotators, same vignettes, **no codebook** — intuition-only classification. Measures the criterion's increment over lay intuition (both agreement difference and verdict difference; several adversarial cases are designed such that the codebook verdict contradicts intuition).
  - **Exploratory panel (optional):** HN volunteers responding to the **post-hash follow-up call** in the launch thread (per decision (f), 2026-07-13, the launch-day first comment carries no recruitment sentence; the call is posted only after the answer-key hash is public, so no enrollment, materials, or eligibility collection can precede the hash); Session A only (~80–100 min); reported separately; never pooled with the verification panel (HN-thread readers fail the exposure screen by definition).
  - **LLM panel (quarantined secondary condition):** ≥3 models from different vendors, versions pinned, temperature 0, 3 samples per model with self-consistency reported, verbatim human codebook, de-branded cases only. Never pooled with human α; interpreted, per pre-registration, as a demonstration adjacent to the paper's §6.3 rented-interpreter argument. Rationale for quarantine: the criterion was itself drafted with LLM assistance (verification ledger §8.2 records the circularity warning), and the handbook — including Table 2 with verdicts — is public with a DOI, so training-data contamination must be assumed.
- **Contamination controls:** vignettes and fact cards avoid the paper's verdict vocabulary; no one-to-one reuse of Table 2 rows; de-branding rule for systems where brand attitudes could dominate judgment; per-annotator randomized item order; minimal-pair members separated in sequence with surface details varied; annotators are isolated (independence statement, staggered scheduling, clarifications only via a written FAQ synchronized to all annotators); a **naive-reader screening rule** is run on every vignette and fact card before the set is frozen: three readers who have not seen the criterion independently guess the "intended" answer per item (forced three-way choice) with a one-line reason; an item is rewritten and re-screened if all three match the author key, or if two match and any reason cites surface wording (phrasing, sentence shape, ending pattern) rather than situational facts — a pre-set screening procedure, not a significance test (no chance model is claimed at this panel size).

## 3 Materials

1. **Codebook** (companion document; the instrument under test; every decision node cites the paper's §2.2 text; explicit decision order resolving the hybrid-vs-tiebreaker precedence; three worked examples on independent material).
2. **Vignettes** (case-set design document; specification and audit trail there).
3. **Fact cards** (Task B): fixed field template — funding source; who can modify objectives/settings; refusal affordances present; what data the system learns from (where publicly documented) — each field filled only with publicly citable evidence (links recorded in the researcher-side evidence ledger; annotator-facing cards are de-branded plain text, per the taskB document §2 design ruling). Fact cards pass the same screening rule. Where a case's *deployment variant* is ambiguous (the facts would differ by variant), it is concretized to one variant or excluded — no "depends" cases in the frozen set. A field whose value is verifiably **not publicly documented** is, by contrast, a legitimate fact-card value, recorded as such with the search trail in the evidence ledger: whether the loyalty traits are judgeable from public information is part of what Task B measures (§6).
4. **Annotation forms** (appendix A of this protocol).
5. **Answer key:** authors' pre-judgments for all cases (three tests, placement, traits). Written when the case set freezes; its SHA-256 hash is published **before recruitment begins** as a public commit in the handbook repository (decided 2026-07-10; an OSF registration is added on top if an account is available before recruitment); plaintext released only with the study data.

## 4 Procedure

1. Registration (two-stage): protocol + answer-key hash registered before the pilot; pilot amendments logged; codebook frozen before main annotation. The §2.2 criterion text version hash is included in the registration; any post-measurement change to the criterion in the paper is disclosed as such.
2. Training: codebook reading + 3 worked examples (~20–30 min, self-paced, logged).
3. Pilot: 5 independent cases (never entering the main set) → written calibration feedback, synchronized to all annotators → codebook amendments (if any) recorded; then freeze.
4. Main annotation: Session A; lock; Session B (verification panel only).
5. Debrief item: annotators may flag cases they found ill-posed (free text; used in the disagreement analysis).

## 5 Statistical analysis plan (pre-registered)

- **Primary endpoints (two):** Krippendorff's α (nominal) for **placement**, computed separately for the representative stratum and the adversarial stratum. Fleiss' κ is reported in a footnote for continuity with older literature only; α is the inferential statistic throughout (handles the fixed-subset structure, any missingness, and ordinal trait scales).
- **Secondary/descriptive:** full-set placement α (computed over the designed composition — **not** an estimate of reliability under any natural category distribution, since the anchor stratum deliberately oversamples production and hybrid acts); α for each of the three output tests; α (ordinal) for each trait T1–T4 (T1 flagged as a consistency check on the information test, not independent evidence); interval-α sensitivity on a 5-point production↔consumption slider collected alongside the categorical choice; leave-one-rater-out diagnostics; answer-key correspondence rate (validity direction, reported separately from reliability); confidence-stratified disagreement analysis (operational definition: high-confidence disagreement = criterion ambiguity; low-confidence disagreement = case difficulty); baseline-panel comparison (agreement and verdict deltas); intra-rater repeat consistency; per-annotator label distributions (hybrid-rate outliers flagged as escape-valve behavior).
- **Hybrid reporting boundary (R1):** the six designed hybrid-gate cases (ANC-08..13) never enter the co-primary endpoints (those cover the representative and adversarial strata only). For each of the six, the study reports: the five annotators' label distribution, whether a valid two-component decomposition was given, rationale type, and answer-key correspondence — and nothing more: **no hybrid-specific α, no p-values, no population-prevalence inference.** Incidental hybrid labels arising inside the representative or adversarial strata remain part of those strata's nominal three-category α. One pre-registered edge ruling: REP-12 (a single conversational session ending in a transaction) is ruled *one act* — the author key records its placement under the generation-time-intent reading, and annotator decomposition of it is analyzed as boundary data, not error.
- **Uncertainty:** bootstrap CIs; algorithm, software package, and version pinned at registration (Krippendorff's own bootstrap vs. naive case bootstrap differ at this n; the choice is recorded). Adversarial-stratum CIs are expected to be wide (±0.15–0.2); its primary product is the qualitative disagreement typology, stated in advance.
- **Exclusion rules (pre-registered):** attention-check failure threshold; minimum plausible response time; handling of incomplete sessions; missingness reported and handled by α's native mechanism. Panel size is fixed at 5; attrition is reported, not replaced mid-study.
- Analysis scripts and anonymized raw data are released with the handbook.

## 6 Interpretation bands and consequence binding (pre-registered)

Bands follow Krippendorff's conventional thresholds (α ≥ 0.80 reliable; 0.667–0.80 tentative), with the applicability argument stated: the criterion's use here is category gating for a research agenda — moderate misclassification cost — so the conventional bands are appropriate. Bands apply **only to the two primary endpoints**; traits are interpreted individually.

Consequences are bound to the paper's own wording in advance:

| Representative-stratum placement α | Obligation on the paper text |
|---|---|
| ≥ 0.80 | current "operational criterion" wording stands |
| 0.667–0.80 | "operational" retained, but the disagreement clusters and the indicated codebook revisions must be reported in the main text |
| < 0.667 | the criterion is demoted to "proposed criterion" throughout; Table 2 carries a reliability caveat |

Pre-registered interpretation for low T3/T4 α: these traits may be unannotatable from public information — which, if observed, is evidence for the paper's own auditability agenda (loyalty must be audited, not read off), not a rescue added post hoc. Any post-data codebook revision is either re-validated on new annotators and cases, or labeled in the paper as an unvalidated revision.

## 7 Ethics

Minimal-risk, exempt-category annotation study of *system descriptions* (annotators judge vignettes, not personal data). No IRB of record (independent researcher); compensating procedural rigor: informed consent covering open-data release of anonymized annotations, withdrawal rights, payment on withdrawal pro-rata; no demographic data beyond coarse background category; consent text stored with the registration. The paper carries a one-sentence exempt rationale. **Consent explicitly licenses the anonymized annotation data for CC0/public-domain-equivalent release** — HDSR requires replication data to be deposited in its Dataverse (default CC0), so the consent must permit that from the start (venue memo, 2026-07-10).

## 8 Outputs

- Paper: ~800–1,200 words in the main text (α table + disagreement typology) + full protocol, codebook, case set, data, and scripts as the online supplement / handbook release (the case set doubles as the seed evaluation set for the third-wave benchmark, per TODO A2.6/B7).
- Timeline: materials (this package) → leakage audit + answer key + registration → recruitment (triggered by HN response wave and/or Prolific, per TODO A2.3) → pilot → two sessions → analysis. End-to-end ≈ 4–7 weeks, running inside the HDSR proposal response window.
